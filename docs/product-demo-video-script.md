# WiseCall Product Demo Video Script

## Recommended Format

- Length: 2-3 minutes
- Style: Screen recording with calm voiceover
- Audience: UK service businesses evaluating an AI receptionist
- Goal: Show that a new customer can start a trial, create an agent from their website, review the setup, then activate phone, knowledge, web chat and email channels

## Demo Setup

Use a clean demo account, not a real customer account.

- Demo business: RinseDental
- Website: `https://rinsedental.example`
- Agent name: Sophie
- Opening hours: Mon-Fri 08:30-17:30, Sat 09:00-13:00, Sun closed
- Default inbox: `reception@rinsedental.example`
- Test routing contact: Reception

## Short Synthesia Version

Use this when the video is avatar-led with screen recordings added as media.

```text
In this demo, we'll set up a WiseCall AI receptionist from scratch.

A new customer starts at app.wisecall.io, signs up with their email, and chooses the plan they want to trial. Every plan includes the AI receptionist, a complete phone system and UK outbound calling, with a 7-day trial so the business can test the product before going live.

Once inside the dashboard, the customer clicks Create agent. WiseCall asks for the business website, then drafts the receptionist automatically: the business details, the greeting, what the agent should say, what it knows, and any opening hours it can find.

The customer reviews everything before it goes live. They can change the business name, choose the receptionist name, edit the greeting, adjust the prompt, and confirm opening hours. Outside those hours, WiseCall can take a detailed message and send it to the team.

After the receptionist is created, the business can fine-tune behaviour, test the voice, add more Knowledge Base content, and decide where urgent calls and summaries should be routed.

WiseCall can then assign a phone number so the AI receptionist starts handling calls. The same agent can also be added to the website as a chat bubble using one line of code, with brand colour controls and a live preview.

For businesses that want email handled too, the Email channel lets them forward an inbox, such as support@, so the same AI replies to emails and logs every contact alongside phone calls.

That is the WiseCall setup flow: choose a trial, create an AI receptionist from a website, review the setup, and add phone, website chat and email channels as the business grows.
```

## Screen Recording Checklist

1. Sign up with a disposable demo email.
2. Choose a trial plan.
3. Create an agent from a website.
4. Review the generated draft.
5. Enter or confirm opening hours.
6. Create the receptionist.
7. Show Behaviour settings.
8. Add a Knowledge Base source.
9. Show routing and office hours.
10. Assign or show phone number.
11. Open Channels.
12. Expand Website chat and copy the embed code.
13. Show Email channel add-on.
14. End on Contacts, Call History or Dashboard.

## Notes For Recording

- Do not show real card details, real customer calls, production API keys or private emails.
- Use a demo account seeded with example calls if possible, so Contacts, Call History and AI Insights are not empty.
- If checkout is live, record up to the Stripe page, then cut to an already-prepared trial account.
- Keep cursor movement slow and deliberate; pause for half a second after each click so the viewer can track the flow.
- Blur email addresses, phone numbers and billing details unless they are clearly fake demo data.
